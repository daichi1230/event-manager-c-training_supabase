# Event Manager C + Supabase コピペ手順

## 1. フォルダへ移動

```powershell
cd <展開したフォルダ>\event-manager-c-supabase
```

## 2. 依存関係を入れる

```powershell
npm install
```

## 3. 環境変数ファイルを作る

```powershell
Copy-Item .env.example .env.local
notepad .env.local
```

`.env.local` に次を入れて保存します。

```env
VITE_SUPABASE_URL=https://YOUR-PROJECT-REF.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=YOUR_SUPABASE_PUBLISHABLE_OR_ANON_KEY
```

## 4. Supabase の SQL を流す

- Supabase Dashboard を開く
- SQL Editor を開く
- `supabase/schema.sql` の中身を丸ごと貼る
- Run を押す

## 5. 開発起動

```powershell
npm run dev
```

開くURL:

```text
http://localhost:5173
```

## 6. サインアップ

- 画面でサインアップ
- 表示名 / メール / パスワードを入力

## 7. 自分を管理者にする

Supabase の SQL Editor で次を実行:

```sql
update public.profiles
set role = 'admin'
where id = (
  select id
  from auth.users
  where email = 'your-admin@example.com'
);
```

## 8. 画面を再読み込み

これで管理者メニューが出ます。

## 9. 本番ビルド確認

```powershell
npm run build
npm run preview
```

開くURL:

```text
http://localhost:4173
```

## 10. GitHubへ push

```powershell
git init
git add .
git commit -m "Add Event Manager C Supabase"
git branch -M main
git remote add origin https://github.com/<GitHubユーザー名>/event-manager-c-supabase.git
git push -u origin main
```

## 11. デプロイ

### Vercel の場合
- Import Project
- GitHub リポジトリ選択
- Environment Variables に `.env.local` と同じ値を登録
- Deploy

### GitHub Pages の場合
- 静的フロント自体は公開可能
- ただし環境変数の扱いとSPAのビルド配置を揃える必要があります
- Vercel の方が簡単です
